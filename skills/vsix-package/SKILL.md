# vsix-package Skill

将 openclaw-code 工作目录打包成 `.vsix` 插件文件。

## 版本号规则

- **默认行为**：每次打包自动递增 patch 版本（0.1.0 → 0.1.1 → 0.1.2...）
- **大版本升级**：只有明确说"升级到 0.2"或"升级到 1.0"时才升级对应主版本或次版本
- 版本号保存在工作目录的 `.version` 文件中

## 触发方式

当用户提到"打包"、"package"、"vsix"、"发布"等关键词时，激活此技能。

## 执行流程

1. 检查 `.version` 文件是否存在，不存在则创建为 `0.1.0`
2. 读取当前版本号
3. 递增 patch 版本（或按指令升级大版本）
4. 更新 `.version` 文件
5. 执行 `vsce package` 打包
6. 输出文件名格式：`openclaw-code-{version}.vsix`

## 使用方式

```
打包
/打包
发布
package
```

## 升级大版本命令

```
升级到 0.2   → 次版本升级（0.1.9 → 0.2.0）
升级到 1.0   → 主版本升级（0.1.x → 1.0.0）
升级到 2.0   → 主版本升级（1.x.x → 2.0.0）
```
